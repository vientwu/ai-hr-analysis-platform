## 问题与原因
- 控制台出现两条日志：`ERR_ABORTED .../logout?scope=local` 与 `TypeError: Failed to fetch`
- 由 Supabase UMD 库在执行 `auth.signOut` 时仍触发网络请求导致（本地预览网络/页面切换中断），即使 `scope: 'local'` 也可能访问接口并在被中断时抛出错误

## 目标
- 点击“退出”不再产生网络错误日志
- 保持 UI 与会话状态正确更新（清除本地会话、切换登录/退出按钮）

## 改动方案
1. 纯本地注销（默认）
- 在 `js/supabase.js` 与 `public/js/supabase.js` 中将 `signOut` 改为本地优先，不调用远程接口：
  - 检测 `navigator.onLine` 或 `window.Auth.mode !== 'supabase'` 时直接本地退出
  - 清理本地会话：`currentUser = null`，派发 `auth-changed`
  - 主动删除 Supabase 本地存储的令牌（遍历 `localStorage` 删除形如 `sb-<project-ref>-auth-token*` 的键）
- 保留一个可选开关（如 `window.__SUPABASE_LOGOUT_SERVER === true`）时才调用远程 `client.auth.signOut({ scope: 'global' })`

2. 容错与静默
- 所有远程退出调用均包裹 `try/catch`，失败时不抛错，仅执行本地清理并打印 `console.debug`
- 避免在预览环境触发跨域或页面卸载中的网络请求

3. UI 行为
- 保持现有“已退出登录”提示与导航切换，不改动 UI 文案

## 验证
- 在预览中点击“退出”，观察控制台不再出现 `logout?...` 的网络错误
- “我的报告”模块显示需登录提示；顶部登录/退出按钮状态正确

## 影响面
- 仅影响前端退出逻辑，不改动登录/保存报告等功能
- 生产环境如需服务端全局登出，可通过开关启用，无需改代码

## 后续可选
- 增加统一的环境配置注入（如 `window.__SUPABASE_CONFIG.logoutMode`），便于按环境开启远程注销
