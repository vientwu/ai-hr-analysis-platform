## 概览
- 项目类型：Node.js 静态前端 + Serverless API（Vercel Functions）+ 本地 Express 备用后端
- 主要端口：前端 `4321`，本地 API `4000`
- 关键文件：`package.json`、`README.md`、`.env(.example)`、`api/*.js`、`dev-server.js`

## 前置条件
- 安装 `Node.js >= 18` 与 `npm`
- 按 `.env.example` 创建 `.env` 并填写：`SUPABASE_URL`、`SUPABASE_ANON_KEY`、`COZE_PAT`（以及可选的 `COZE_RESUME_WORKFLOW_ID`、`COZE_INTERVIEW_WORKFLOW_ID`、`API_DEV_PORT`）
-（可选）安装 Vercel CLI：`npm i -g vercel` 并 `vercel login`

## 安装依赖
- 在项目根目录执行：
  - `npm install`

## 启动前端（开发）
- 在项目根目录执行：
  - `npm run dev`
- 访问：`http://localhost:4321`
- 若未使用 `public/index.html`，开发环境会兜底加载根目录 `index.html`

## 启动本地 API（联调可选）
- 确认 `.env` 已配置（由 `dotenv` 自动读取）
- 在项目根目录执行：
  - `npm run api:dev`
- 验证：
  - `http://127.0.0.1:4000/api/status`
  - `http://127.0.0.1:4000/api/health`

## 前后端联调说明
- 开发态下前端脚本会将请求自动指向 `http://127.0.0.1:4000/api/...`
- 建议同时启动前端与本地 API，以完整体验分析功能

## 生产部署（可选）
- 使用 Vercel：
  - `vercel`（首次需登录并选择项目）
  - 部署默认以 `public/` 为静态资源目录，API 位于 `api/*.js`

## 验证清单
- 前端页面可正常打开，无控制台报错
- 触发“分析”操作时网络请求命中 `api/resume-analyze` 或 `api/interview-analyze` 并返回 200/成功数据
- `api/status`、`api/health` 均返回正常结果

## 常见问题与处理
- 端口占用：通过 `.env` 设置 `API_DEV_PORT`，或调整前端端口（若命令支持）
- 环境变量缺失：API 返回 401/403；请补齐 `.env` 中的必填项
- Windows 中文路径：建议在项目根直接执行命令，避免额外路径转义
